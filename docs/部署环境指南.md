# TradingStudio 部署环境指南

> 面向个人期货交易的真实部署方案。
> 约束：C# + CTP + 中低频 + 7×24 运行 + 成本可控

---

## 1. 一句话部署方案

**一台 2C4G Windows 云服务器，跑 .NET 应用 + PostgreSQL，通过 CTP 接入行情和交易。**

```
┌─────────────────────────────────────────┐
│              Windows Server             │
│                                         │
│  ┌──────────┐  ┌──────────┐            │
│  │ MdService │  │ TdService │  .NET 10  │
│  │ (行情采集) │  │ (交易执行) │  Windows  │
│  │  7×24    │  │  按需    │   Service  │
│  └────┬─────┘  └────┬─────┘            │
│       │              │                  │
│  ┌────▼─────┐  ┌────▼─────┐            │
│  │ Tick 文件 │  │PostgreSQL│            │
│  │ .tick    │  │ pg 17    │            │
│  └──────────┘  └──────────┘            │
│                                         │
│  ┌──────────────────────────┐          │
│  │      WPF 监控面板         │          │
│  │   (本地远程桌面连接)       │          │
│  └──────────────────────────┘          │
└─────────────────────────────────────────┘
         │                    │
         ▼                    ▼
    CTP MdApi            CTP TraderApi
  (tcp://broker)       (tcp://broker)
```

## 2. 硬件配置

### 2.1 推荐配置

| 项目 | 最低 | 推荐 | 说明 |
|------|------|------|------|
| CPU | 2 核 | 4 核 | 2 核够用，4 核留余量做回测 |
| 内存 | 2 GB | 4 GB | 实际使用 < 200 MB，但 Windows 本身吃内存 |
| 系统盘 | 40 GB SSD | 60 GB SSD | Windows + .NET runtime + PG |
| 数据盘 | 60 GB SSD | 100 GB SSD | Tick 文件 + PostgreSQL 数据 |
| 网络 | 1 Mbps | 5 Mbps | CTP 流量 < 100 KB/s，够用 |

### 2.2 存储计算

基于数据实测：

| 数据 | 每天 | 每年 |
|------|------|------|
| Tick 文件（10 个活跃品种） | ~38 MB | ~9.5 GB |
| Tick 文件（全部 74 品种） | ~280 MB | ~70 GB |
| Bar 数据（PG，1min） | ~5 MB | ~1.3 GB |
| PostgreSQL 元数据 | — | < 100 MB |
| 日志文件（JSON Lines） | ~2 MB | ~500 MB |
| **合计（10 活跃品种）** | **~45 MB/天** | **~12 GB/年** |

> 100 GB 数据盘可用 **5-8 年**。不够了随时扩容云盘。

### 2.3 实测数据（doc 01 性能分析）

```
CPU:  平均 < 5%（100 tick/s 全市场）
内存:  < 200 MB（含 2000 Bar 窗口 + 10 策略）
网络:  < 100 KB/s
磁盘:  8 KB/s 写入（Tick），每分钟 Bar 闭合时几十 KB
```

**结论：2C4G 绰绰有余。**

## 3. 软件清单

### 3.1 必须安装

| 软件 | 版本 | 用途 | 安装方式 |
|------|------|------|----------|
| Windows Server | 2019+ 或 Win10+ | 操作系统 | 云厂商镜像 |
| .NET Runtime | 10.0 | 运行 C# 应用 | `winget install Microsoft.DotNet.Runtime.10` |
| PostgreSQL | 17.x | Bar + 元数据 + 交易记录 | `winget install PostgreSQL.PostgreSQL` |
| CTP DLL | SDK 6.7.13+ | 行情和交易接口 | 复制到应用目录 |

### 3.2 可选安装

| 软件 | 用途 | 何时需要 |
|------|------|----------|
| IIS / Nginx | 健康检查 HTTP 端点 | 外部监控时 |
| pgAdmin | PG 管理工具 | 调试数据库时 |
| 远程桌面 | 日常管理 | 云服务器标配 |

### 3.3 不需要的

| 软件 | 为什么不需要 |
|------|-------------|
| Redis | 进程内内存缓存已足够 |
| ClickHouse / TDengine | Tick 文件 + PG 够用，中低频不需要时序数据库 |
| Docker | 增加复杂度，Windows + Docker = 额外折腾 |
| 消息队列 (RabbitMQ/Kafka) | Channel<T> 进程内队列已足够 |
| Prometheus / Grafana | 日志 + 健康检查端点够用 |

## 4. 网络需求

### 4.1 CTP 连接

```
行情 (MdApi):
  tcp://180.168.146.187:10211  (SimNow 模拟)
  tcp://实盘前置机地址:端口       (期货公司提供)

交易 (TraderApi):
  tcp://实盘交易前置机:端口       (期货公司提供，通常与行情不同)
```

**要求：**
- 服务器能出站访问期货公司 CTP 前置机（TCP）
- 不需要公网 IP（出站连接即可）
- 带宽要求极低（< 100 KB/s）
- 延迟不敏感（秒级足够）

### 4.2 管理访问

| 访问方式 | 端口 | 用途 |
|----------|------|------|
| 远程桌面 (RDP) | 3389 | 日常管理、WPF 监控面板 |
| HTTP 健康检查 | 5000 | 外部存活检测 |
| PostgreSQL | 5432 | 数据库管理（建议仅本地监听） |

## 5. 运维要点

### 5.1 7×24 运行保障

```
┌─────────────────────────────────────────────┐
│  行情采集服务 (MdService)                     │
│  - Windows Service，开机自启                  │
│  - 崩溃自动重启（SC Failure Actions）         │
│  - CTP 断线自动重连（应用层实现）              │
│  - 交易日历驱动：开盘前自动启动订阅            │
│  - 凌晨 2:30-8:30 自然空闲（无行情）          │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  交易服务 (TdService)                         │
│  - 只在策略需要交易时运行                      │
│  - 手动启动 / 策略条件触发                    │
│  - 登录后保持心跳（CTP 要求）                  │
│  - 风控硬编码，关机前必须撤销所有挂单          │
└─────────────────────────────────────────────┘
```

### 5.2 关键文件路径

```
D:\TradingStudio\
├── app\                          ← 应用程序
│   ├── TradingStudio.Data.exe
│   ├── TradingStudio.Execution.exe
│   ├── thostmduserapi_se.dll     ← CTP 行情 DLL
│   ├── thosttraderapi_se.dll     ← CTP 交易 DLL
│   ├── TradingStudioCtpBridge.dll ← 自建 C++ 桥接
│   └── appsettings.json          ← 配置文件
│
├── data\
│   ├── ticks\                    ← Tick 文件存储
│   │   └── {symbol}\
│   │       └── {tradingDay}_{symbol}.tick
│   └── specs\                    ← 合约规格缓存
│       └── contracts.json
│
├── logs\                         ← 结构化日志（JSON Lines）
│   ├── mdservice-{date}.jsonl
│   └── tdservice-{date}.jsonl
│
└── backup\                       ← PostgreSQL 定时备份
    └── tradingstudio-{date}.sql.gz
```

### 5.3 备份策略

| 内容 | 频率 | 方式 |
|------|------|------|
| PostgreSQL 数据库 | 每天凌晨 3:00 | `pg_dump` → 压缩 → 本地 + 异地 |
| Tick 文件 | 实时 | 源数据，丢失不可恢复（CTP 不提供历史） |
| 日志文件 | 保留 30 天 | 自动轮转，过期删除 |
| 合约规格 | 每周 | 从 AKShare 拉取刷新 |

### 5.4 监控

```
健康检查端点: GET http://localhost:5000/health
  → 200 OK: 一切正常
  → 503: 异常（附详情）

检查项:
  ├── CTP 行情连接状态
  ├── CTP 交易连接状态
  ├── PostgreSQL 连接
  ├── Tick 文件写入延迟
  ├── 内存使用
  └── 磁盘剩余空间
```

> 用手机 App（如 Server酱 / Pushover）接收告警。不建监控大盘——个人用，手机通知足够。

## 6. 成本估算

### 6.1 月费方案

| 方案 | 配置 | 月费 | 适用阶段 |
|------|------|------|----------|
| 轻量云服务器 | 2C2G 40GB | ~60 元 | 开发测试 |
| **标准云服务器** | **2C4G 100GB** | **~100 元** | **实盘运行** |
| 轻量云服务器（大） | 4C8G 150GB | ~200 元 | 实盘 + 回测并行 |

### 6.2 替代方案：家用服务器

```
一次性投入: ~2000 元（迷你主机，如 Intel N100 16GB+512GB）
月费: 电费 ~10 元/月
缺点:
  - 家庭网络不稳定（断网 = 断行情）
  - 停电 = 系统中断
  - 没有固定公网 IP（通常不需要）
优点:
  - 零月费
  - 数据完全在本地
```

> **建议：** 实盘阶段用云服务器（网络稳定）。回测开发阶段随便用什么机器。

## 7. 环境搭建步骤（概览）

```powershell
# 1. 安装 .NET Runtime
winget install Microsoft.DotNet.Runtime.10

# 2. 安装 PostgreSQL
winget install PostgreSQL.PostgreSQL
# 初始化数据库，创建 tradingstudio 库，执行 DDL (doc 10)

# 3. 复制应用程序文件
# 从开发机发布: dotnet publish -c Release -r win-x64 --self-contained
# 复制 publish/ 到服务器 D:\TradingStudio\app\

# 4. 复制 CTP DLL
# thostmduserapi_se.dll, thosttraderapi_se.dll → app/

# 5. 配置 appsettings.json
# {
#   "Ctp": { "MdFront": "tcp://...", "TdFront": "tcp://...",
#            "BrokerId": "9999", "UserId": "...", "Password": "..." },
#   "Data": { "TickPath": "D:\\TradingStudio\\data\\ticks" },
#   "PgConnection": "Host=localhost;Database=tradingstudio;..."
# }

# 6. 注册 Windows Service
New-Service -Name "TradingStudio.Md" `
  -BinaryPathName "D:\TradingStudio\app\TradingStudio.Data.exe" `
  -StartupType Automatic

# 7. 启动
Start-Service TradingStudio.Md
```

## 8. 常见问题

**Q: 为什么不用 Docker？**
Windows + Docker = 额外复杂度。.NET 自带独立发布（self-contained），一个文件夹拷贝过去就能跑。比 Docker 少一层抽象。

**Q: 为什么不用 Linux？**
CTP DLL 是 Windows 原生 DLL（不可能在 Linux 跑）。虽然有 CTP 的 Linux SDK，但 Windows 是你舒适区。

**Q: 云服务器断网怎么办？**
CTP 行情断了 = 数据缺口。应用层检测断线后自动重连；缺口的数据只能在恢复后用"前向填充"补齐 K 线（tick 数据无法回补）。关键是：断网期间有没有持仓？→ 有持仓时加手机告警。

**Q: 需要固定 IP 吗？**
不需要。CTP 是出站 TCP 连接，服务器主动连期货公司前置机。但如果你要远程桌面进去管理，云服务器自带公网 IP。

**Q: PostgreSQL 在哪台机器？**
同一台服务器。本地连接（localhost），零网络开销。备份额外存一份到其他地方。
