# TradingStudio WPF 监控客户端 — 设计方案 v2

> 基于 [WPF量化交易界面设计](./WPF量化交易界面设计.md) 参考文档，结合现有引擎代码。
> AvalonDock 工作台模式 | OxyPlot 深色主题 | MVVM

---

## 1. 整体架构

```
TradingStudio.UI.exe                    TradingStudio.exe (localhost:5199)
        │                                        │
        │  SignalR Client                         │  SignalR Hub
        │  REST API                               │  REST API
        │                                         │
        ▼                                         ▼
   WPF 监控客户端                           实盘引擎 (7×24)
   ┌─────────────────────┐                ┌──────────────────────┐
   │ AvalonDock 工作台    │◄──────────────│ TradingEngine        │
   │ 左侧: 导航面板       │  SignalR推送  │  PortfolioManager    │
   │ 中央: Document工作区 │  (实时行情,   │  StrategyContainer   │
   │ 右侧: 告警面板       │   订单状态,   │  ExecutionHandler    │
   │ 底部: 日志中心       │   告警)       │  FeedbackMonitor     │
   └─────────────────────┘                └──────────────────────┘
```

**核心原则**（从参考文档）:
- UI 是临时访客，引擎是主人 — 断开不影响引擎
- AvalonDock 支持多屏、拖拽、布局保存恢复
- 左侧导航 + 中央多 Document + 右侧告警 + 底部日志

## 7. 回测执行流（客户端内嵌引擎）

```
用户填配置 → 点 [运行]
  │
  ▼
BacktestRunner.RunAsync(config)
  │  (后台 Task, 不阻塞 UI)
  │
  ├── new BarStore(dbPath)
  ├── new HistoricalBarFeed(periodMinutes)
  ├── new ExecutionHandler + PortfolioManager
  ├── StrategyFactory.Create → 策略实例
  └── new TradingEngine(...).RunAsync(ct)
         │
         ▼
  EngineReport { EquityCurve, Trades, PerformanceReport }
         │
         ▼
  Dispatcher.Invoke → 更新图表 + 数据表
```

客户端内嵌引擎——回测在本地执行，不需要服务器。

## 9. 分阶段实施

### Phase A: AvalonDock 骨架 + 总览 (1-2 天)

- 引入 AvalonDock NuGet 包
- ShellViewModel + ShellView (左侧/中央/右侧/底部)
- DashboardViewModel: KPI 卡片 + 权益曲线
- 对接 `/api/health` + `/api/portfolio`

### Phase B: 策略监控 + 告警 (1-2 天)

- StrategyMonitorViewModel: K线 + 挂单 + 成交
- AlertPanelViewModel: SignalR 告警推送
- 对接 `/api/strategies/*`

### Phase C: 回测分析 (2-3 天)

- BacktestViewModel: 配置表单 + 参数反射
- BacktestRunner: 内嵌引擎执行
- 权益曲线 + 交易标记图表
- 绩效表 + 参数对比

### Phase D: 完善 (1-2 天)

- 日志中心 + 订单流
- 布局保存/恢复 (XmlLayoutSerializer)
- 多显示器浮动窗口

## 10. 关键决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 图表库 | OxyPlot（已有） | 现有代码复用，深色主题一致 |
| 布局框架 | AvalonDock | 参考文档指定，多 Document + 拖拽 |
| 回测位置 | 客户端内嵌引擎 | 无需服务器，本地秒级执行 |
| 数据连接 | SignalR + REST | Phase 3 已定义，引擎自带 |
| MVVM 框架 | CommunityToolkit.Mvvm（已有） | 源码生成，代码量少 |
