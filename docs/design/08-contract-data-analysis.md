## 二、字段完整性

所有 76 份规格文档结构统一，16 个字段中 14 个覆盖率 100%：

| 字段 | 覆盖率 | 说明 |
|---|---|---|
| 交易品种/交易单位/报价单位/最小变动价位 | 100% | 所有品种必有 |
| 涨跌停板幅度/合约月份/交易时间 | 100% | — |
| 最后交易日/交割日期/交割品级 | 100% | — |
| 最低交易保证金/交割方式/交易代码/上市交易所 | 100% | — |
| 交割地点 | 88% | CFFEX（现金交割）和 INE 部分品种无此字段 |
| 交割单位 | 20% | 仅 SHFE 有色金属+黑色金属有此字段 |

**结论：** 数据模型可以为所有品种设计统一 schema，可选字段用 NULL。

## 三、关键发现：六大维度交叉对比

### 3.1 交易单位 — 8 种不同单位体系

| 单位 | 品种数 | 典型品种 |
|---|---|---|
| 吨 | 62 | 绝大部分商品期货 |
| 指数点（每点×元） | 4 | 四大股指（IF/IC/IH/IM） |
| 面值（百元报价×万元） | 4 | 国债（TS/TF/T/TL） |
| 立方米 | 2 | 纤维板、原木 |
| 克 | 1 | 黄金 |
| 千克 | 1 | 白银 |
| 桶 | 1 | 原油 |
| 元/点 | 1 | SCFIS欧线（50元/点） |

**数据模型含义：** `trading_unit` 不能用单一数字类型——需要同时存储数值和单位。CFFEX 的"每点300元"说明有些品种的交易单位不是一个简单的质量/体积数。

### 3.2 最小变动价位 — 0.005 到 50，跨度 10,000 倍

| Tick Size | 品种数 | 典型品种 |
|---|---|---|
| 0.005 | 3 | 2Y/5Y/10Y 国债 |
| 0.01 | 1 | 30Y 国债 |
| 0.02 | 1 | 黄金（元/克） |
| 0.1 | 2 | SCFIS欧线、原油 |
| 0.2 | 4 | 四大股指期货 |
| 0.5 | 5 | 焦炭/焦煤/铁矿石/纤维板/原木 |
| 1 | 29 | 螺纹钢/热卷/玉米/豆粕/PTA/甲醇... |
| 2 | 10 | 豆油/棕榈油/硅铁/锰硅... |
| 5 | 14 | 铝/锌/铅/橡胶/棉花/生猪... |
| 10 | 4 | 铜/镍/锡/铜(BC) |
| 50 | 1 | 碳酸锂 |

**数据模型含义：** `tick_size` 需要 `NUMERIC(12,6)` 精度——0.005 要求至少 3 位小数。`NUMERIC(12,6)` 可以覆盖从 0.005（国债）到 50（碳酸锂）的完整范围。

### 3.3 涨跌停板 — 0.5% 到 10%

| 幅度 | 品种数 | 说明 |
|---|---|---|
| ±0.5% | 1 | 2年期国债 |
| ±1.0% ~ ±3.5% | 3 | 国债期货 |
| ±3% | 14 | SHFE 大部分品种 |
| ±4% | 42 | DCE、CZCE 大部分品种 |
| ±5% | 8 | 锡/燃油/沥青/丁二烯胶/低硫燃油 等 |
| ±8% ~ ±10% | 8 | 燃料油/SCFIS欧线/四大股指 |

**数据模型含义：** `price_limit` 是百分比，存储为 `NUMERIC(5,3)`（如 0.030、0.100）。

### 3.4 保证金率 — 0.5% 到 12%

| 保证金率 | 品种数 |
|---|---|
| 0.5% ~ 2% | 3（国债） |
| 3.5% ~ 4% | 4（股指+白银+沥青） |
| 5% | 61（绝大多数商品期货） |
| 7% ~ 8% | 3 |
| 12% | 5（SCFIS欧线+四大股指） |

**数据模型含义：** 保证金率像阶梯分布——国债 < 商品 < 股指/欧线。`margin_rate` 用 `NUMERIC(5,3)`。

### 3.5 合约月份 — 12 种不同模式

| 模式 | 品种数 | 说明 |
|---|---|---|
| 1～12月 | 50 | 连续月份，最简单 |
| 1/3/5/7/9/11 | 9 | DCE 农产品常见模式 |
| 1/3/5/7/8/9/11/12 | 2 | 豆粕/豆油 |
| 最近三个季月 | 4 | 国债期货 |
| 当月+下月+两个季月 | 4 | 股指期货 |
| 其他 | 7 | 苹果(1/3/4/5/10/11/12)、橡胶(1/3/4/5/6/7/9/10/11)等 |

**数据模型含义：** 不能把合约月份简单存为"1-12"布尔数组。需要存为**字符串规则**或**月份列表**。

### 3.6 交割方式 — 实物 vs 现金

| 方式 | 品种数 | 说明 |
|---|---|---|
| 实物交割 | 71 | 所有商品期货 |
| 现金交割 | 5 | CFFEX 四大股指 + SCFIS欧线 |

**数据模型含义：** `delivery_type` 是区分 CFFEX 与商品期货的核心字段。现金交割品种没有交割品级/交割地点/交割单位。

## 六、对数据模型的推断

基于以上分析，doc 05 的模型需要以下调整：

1. **`symbols.trading_unit` 字段需要支持复杂单位** — 不能只存数字，需要 `trading_unit_value NUMERIC(12,4)` + `trading_unit_name VARCHAR(50)` 两个字段
2. **`symbols.tick_size` 用 `NUMERIC(12,6)`** — 已正确
3. **`symbols.contract_months` 存字符串** — 不是数组，"1/3/5/7/9/11" 这种模式直接存
4. **新增 `symbols.delivery_type`** — VARCHAR(4): 'PHYSICAL' | 'CASH'
5. **新增 `symbols.product_category`** — 有色金属/黑色/农产品/化工/能源/贵金属/股指/利率/新能源
6. **新增 `symbols.delivery_unit_value` + `delivery_unit_name`** — 区别于 trading_unit
7. **`symbols.code` 允许大小写混合** — CFFEX 用大写 'IF'/'IC'，其他用小写
