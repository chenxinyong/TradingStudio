### 🔴 Bug #2: CTP 断线检测失效 (P0)

**文件:** `src\TradingStudio\Services\CollectService.cs:134-138, 163-167`

```csharp
// OnFrontConnected 先触发 → Task 标记为已完成
md.OnFrontConnected += () => {
    connected.TrySetResult(true);   // Task 状态: RanToCompletion
};

// 断线时 TrySetResult(false) 是 NO-OP! Task 已完成, 无法再次设置
md.OnFrontDisconnected += r => {
    connected.TrySetResult(false);  // ← 返回 false, 静默失败
    loggedIn.TrySetResult(false);   // ← 同上
};

// 断线检测永远不触发!
while (_scheduler.IsInSession())
{
    await Task.Delay(1000, ct);
    // connected.Result 永远是 true → 永远不会 break
    if (connected.Task.IsCompleted && !connected.Task.Result) break;
}
```

**后果:**
- CTP 断开后进程不会有任何反应
- 不会触发外层重连逻辑 (line 92-107)
- 进程变成"僵尸"——等待 `IsInSession()` 返回 false 才退出
- 无法区分"网络断开"和"行情稀疏"两种状态
