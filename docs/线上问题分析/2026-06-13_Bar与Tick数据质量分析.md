## 2. Bar 聚合逻辑 Bug

### 🔴 Bug #4: 1min Bar Volume = Tick 计数，不是成交量

**文件:** `src/TradingStudio.Data/Aggregation/BarAggregator.cs:124`

```csharp
// Accumulate():
bar.Volume++;  // ← 每 Tick +1，丢失真实成交量!
bar.Turnover += 0;  // ← 明确写死为 0，注释说"delta tracked separately"但从未实现
```

**验证数据:**
```
bars_1min: Vol==TickCount = 0/52496   ← 没有一个 Bar 的 Volume 等于 TickCount
             Vol!=TickCount = 52496/52496  ← 全部不等 (Volume 比 TickCount 少 1)
bars_1min: Turnover=0 = 52496/52496   ← 100% 成交额为 0
```

**根因:** `NewBar()` 中 `Volume=0, TickCount=1`，然后每次 `Volume++, TickCount++`，导致 Volume 永远比 TickCount 少 1。且 Volume 记录的是"这一分钟收到了几个 Tick"，完全不是 CTP 的真实成交量（`TickRecord.Volume`）。

### 🔴 Bug #5: Day Bar Volume = Tick 计数，成交额不更新

**文件:** `src/TradingStudio.Data/Aggregation/DailyBarAggregator.cs:45-46, 93-107`

```csharp
// Feed():
bar.Volume++;  // ← 同上，每 Tick +1

// NewDay():
Volume = 1,  // ← 首次设为 1
Turnover = tick.Turnover,  // ← 仅首次赋值，后续不更新!
```

**验证数据:**
```
bars_day: Vol==TickCount = 1072/1072  ← 全部相等（因为 NewDay 时 Volume=1, TickCount=1）
```

虽然 Vol==TickCount 恰好相等（因为起始值一致），但 **Volume 仍然是 Tick 数量，不是成交量**。且 Turnover 只在首 Tick 赋值，后续价格变动后的成交额不更新。

## 3. CSV vs Bars 数据流对比

```
CTP OnQuote ──→ HandleQuote()
                  │
                  ├─→ agg1Min.Feed()  ──→ bars_1min ✅ 546/546 合约有数据
                  ├─→ aggDay.Feed()   ──→ bars_day  ✅ 546/546 合约有数据
                  └─→ tickWriter.Write() ──→ CSV      ❌ 1/546 合约有数据
```

### 06-15 夜盘 CSV 写入失败分析

| 交易所目录 | 06-15 CSV 文件数 | 非空 | 空(0KB) |
|-----------|-----------------|------|---------|
| DCE | 320 | **1** (ag2608) | 319 |
| SHFE | 96 | 0 | 96 |
| INE | 54 | 0 | 54 |
| GFEX | 35 | 0 | 35 |
| CFFEX | 24 | 0 | 24 |
| CZCE | **0** (目录不存在) | 0 | 0 |

**现象：**
- 夜盘开始时（~20:30）创建了 529 个 0KB 空文件
- StreamWriter 已创建、header 已写入缓冲区，但 **从未 Flush 到磁盘**
- 唯独 ag2608 的文件正常写入（1079 行，最后写入 23:00:09）

**可能原因：**
1. `TickCsvWriter.Write()` 内 `lock(entry.Writer)` 抛出异常 → 被 `HandleQuote` 的 try-catch 静默捕获
2. `FlushAll()` 定时器（10 秒）在遍历 `_writers` 时异常 → 仅部分 Writer 被 flush
3. `q.TradingDay` 或 `q.ExchangeID` 字段导致路径异常
4. 文件系统权限/句柄耗尽 (529 个并发 StreamWriter)

**诊断方向：** 周一实时运行时加计数器对比 `_written` vs BarAggregator 的 tick 计数，定位 CSV 写入失败点。
