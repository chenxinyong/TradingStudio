## 1. 总体概要

| 维度 | 日盘 (交易日 06-12) | 夜盘 (交易日 06-15) |
|------|---------------------|---------------------|
| 预期时段 | 08:30–15:30 (7h) | 20:30–02:30 (6h, 白银) |

### Bug 详情

#### #1: GuessExchange 交易所路由

```csharp
// 旧代码:
if ("abcdeijlmprvy".Contains(inst[0])) return "DCE"; // 'a'误伤ag/al/au/ao, 'b'误伤bu/br...
return "SHFE";

// 修复: SHFE 18 前缀显式匹配置于 DCE 通配之前
```

**误伤品种:** ag, al, au, ao, bu, br, cu, pb, rb, ru → CSV 写入 `TickData/DCE/` 而非 `SHFE/`

#### #2: CTP 断线检测

```csharp
// 旧: OnFrontDisconnected 中 TrySetResult(false) 是空操作 (Task 已完成)
// 修复: lock + bool disconnected 标志, 主循环每秒检查
```

#### #3/#4: Bar Volume/Turnover

```csharp
// 旧: bar.Volume++ (Tick计数), bar.Turnover += 0 (写死), Volume 永远≠真实成交量
// 修复: Volume = tick.Volume - lastCumVol (CTP 累计成交量 delta)
//       Turnover = tick.Turnover - lastCumTO (累计成交额 delta)
```

验证: 旧 bars.db 中 52,496 个 1min bar 的 Volume 全部不等于 TickRecord 成交量, Turnover 100% 为 0。

#### #5: 崩溃日志

```csharp
// Program.cs 新增:
AppDomain.CurrentDomain.UnhandledException → crash.log
TaskScheduler.UnobservedTaskException → crash.log
```

#### #6: CSV 写入失败

```
现象: 夜盘 529 个文件创建但未写入, 仅 ag2608 正常
修复: double-check locking + try-catch 隔离 + _failed 字典 + ErrorCount 计数器
根因: 待周一实时验证 (可能: StreamWriter 异常被静默吞掉)
```
