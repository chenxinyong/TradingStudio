# Bar 生成逻辑验证报告

> 验证时间: 2026-06-13
> 验证范围: BarAggregator (1min) + DailyBarAggregator (日线) + SessionScheduler 调度匹配

---

## 1. 1min Bar 生成逻辑

### 1.1 流程

```
Tick → RoundToMinute(tick.ExchangeTime)
        │
        ├── 同分钟 → Accumulate(OHLC更新 + Vol/Turnover delta)
        │
        └── 分钟切换 → Emit旧Bar + NewEmpty()空壳 + Accumulate首Tick
              │
CheckTimeouts(30s): (now - LastTickTime) > 30s → Emit + Reset
收盘: Flush() → 强制 Emit 所有未完成 Bar
```

### 1.2 验证

| 场景 | 行为 | 判定 |
|------|------|------|
| 正常连续交易 | 每分钟切换时自动 Emit | ✅ |
| 盘中休盘 (10:15-10:30) | 无 Tick → 30s timeout → Emit + Reset | ✅ |
| 午休 (11:30-13:30) | 同上 | ✅ |
| 收盘 | Flush() 强制 Emit 最后的未完成 Bar | ✅ |
| 流动性极差 (>30s 无 Tick) | timeout Emit → Reset → 同分钟新 Tick 创建新 Bar | 🟡 同一分钟可能出两根 Bar (低频品种边缘 case) |
| CTP 时间 vs 本地时间 | `RoundToMinute(tick.ExchangeTime)` 用交易所时间 | ✅ |

### 1.3 Volume/Turnover

```
首Tick: Volume=0, Turnover=0, TickCount=1
        _lastCumVol = tick.Volume (CTP 累计)
        _lastCumTO  = tick.Turnover

后续Tick: dV = tick.Volume - _lastCumVol  →  bar.Volume += dV
         dTO = tick.Turnover - _lastCumTO →  bar.Turnover += dTO
         _lastCum* 更新为本Tick累计值
```

| 检查 | 结果 |
|------|------|
| Volume 是否为真实成交量 delta | ✅ |
| Turnover 是否为真实成交额 delta | ✅ |
| 跨日/跨分钟的累计基线正确重置 | ✅ |
| 首 Tick delta 计入 (Volume从0开始) | ✅ |

## 2. 日线生成逻辑

### 2.1 流程

```
Tick → DailyBarAggregator.Feed()
        │
        ├── 首Tick (该合约+交易日) →
        │     NewDay(Vol=0, TO=0), _lastCum[key] = 基线
        │
        └── 后续Tick →
              lock(per-key) {
                OHLC 更新
                dV = tick.Volume - lastCum.Vol → bar.Volume += dV
                dTO = tick.Turnover - lastCum.TO → bar.Turnover += dTO
                _lastCum 更新
              }

收盘: FlushAll() → 全部日线一次性 INSERT OR REPLACE → _bars.Clear()
```

### 2.2 修复: 删除 DayFlushLoop

**原代码 (Bug):**
```
DayFlushLoop: 每60秒 → FlushAll() → INSERT OR REPLACE → _bars.Clear()
→ 日线只保留最后60秒的片段！
```

**修复后:**
```
日线仅在 Session 结束时 FlushAll 一次
→ 全交易日累计，完整落盘
```

### 2.3 隐患: 进程崩溃时未完成的日线丢失

当前日线只在 session 正常结束时落盘。如果进程在 session 中途崩溃：
- 1min bar: 已由 CheckTimeouts + BarStore 批量写入 → 大部分已落盘 ✅
- 日线: 仍在内存中，全部丢失 ❌

**后续优化:** 加 `FlushSnapshot()` — 写入但不 Clear，兼顾实时持久化和数据完整性。

## 3. 调度时段 vs 品种实际交易时间

### 3.1 当前统一调度

```
SessionScheduler:
  日盘: 08:30 - 15:30
  夜盘: 20:30 - 03:00 (次日)
```

### 3.2 各品种差异对比

| 类别 | 实际时间 | SessionScheduler | 影响 |
|------|---------|-----------------|------|
| 商品日盘 | 09:00-10:15, 10:30-11:30, 13:30-15:00 | 08:30-15:30 | ✅ 早30min连/晚30min断, 无 Tick 时段无 Bar |
| CFFEX 股指 | 09:30-11:30, 13:00-15:00 | 同上 | ✅ 时段偏差不影响 |
| CFFEX 国债 | 09:30-11:30, 13:00-15:15 | 同上 | ✅ |
| 夜盘至 23:00 | 21:00-23:00 | 20:30-03:00 | ✅ 超时 4h 空窗 (无Tick) |
| 夜盘至 01:00 | 21:00-01:00 | 同上 | ✅ 超时 2h |
| 夜盘至 02:30 | 21:00-02:30 | 同上 | ✅ 超时 0.5h |
| 无夜盘品种 | 仅日盘 | 仍连夜盘 | ✅ CTP 不发 Tick, 无 Bar |

**结论: 统一调度 = 全时段超集覆盖, 对 Bar 生成无负面影响。**
